# Contributing

## Development setup

1. Install dependencies with npm install.
2. Run the test suite with npm test.
3. Build the TypeScript project with npm run build.

## Working conventions

- Keep changes focused and easy to review.
- Prefer small, well-tested increments.
- Preserve the current MCP and HTTP interfaces unless the change explicitly requires a breaking update.
- Add tests for any behavioral change.

## Pull request expectations

- Include a clear summary of the change.
- Mention any new risks or migrations.
- Ensure tests and build checks pass before opening a PR.
